## 5. 기능 요구사항

### 5.1 Todo 생성

- 사용자는 Todo 제목을 입력할 수 있어야 한다.
- 제목은 필수값이다.
- 생성 직후 기본 상태는 `진행 중`이다.
- 선택적으로 메모, 마감일, 우선순위, 태그를 입력할 수 있다.

### 5.2 Todo 조회

- 사용자는 Todo 목록을 볼 수 있어야 한다.
- 기본 화면은 `오늘` 또는 `전체` 중 하나로 정의한다.
- 각 Todo는 제목, 완료 여부, 마감일, 우선순위를 표시한다.
- 완료된 Todo는 시각적으로 구분한다.

### 5.3 Todo 수정

- 사용자는 기존 Todo의 제목을 수정할 수 있어야 한다.
- 사용자는 메모, 마감일, 우선순위, 태그를 수정할 수 있어야 한다.
- 수정 내용은 저장 후 즉시 목록에 반영되어야 한다.

### 5.4 Todo 완료 처리

- 사용자는 Todo를 완료 또는 미완료 상태로 변경할 수 있어야 한다.
- 완료 시 완료 시간이 기록될 수 있다.
- 완료된 Todo는 기본 목록에서 계속 보일지 숨길지 정책을 정한다.

### 5.5 Todo 삭제

- 사용자는 Todo를 삭제할 수 있어야 한다.
- 삭제 전 확인창을 띄울지 정책을 정한다.
- 삭제 복구 기능 제공 여부를 정한다.

### 5.6 필터 및 정렬

- 사용자는 상태별로 Todo를 필터링할 수 있어야 한다.
  - 전체
  - 진행 중
  - 완료
- 사용자는 다음 기준으로 정렬할 수 있다.
  - 생성일
  - 마감일
  - 우선순위

### 5.7 검색

- 사용자는 Todo 제목 또는 메모에서 키워드 검색을 할 수 있어야 한다.
- 검색 결과는 입력과 동시에 갱신할지, 검색 버튼으로 실행할지 정한다.

### 5.8 알림

- 알림 필요 여부:
- 알림 채널:
  - 앱 내부 알림
  - 브라우저 푸시

- 알림 조건:
  - 마감 전 N분

  - 반복 Todo
